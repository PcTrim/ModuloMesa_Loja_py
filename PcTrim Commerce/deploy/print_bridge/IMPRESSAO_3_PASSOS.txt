CADA PC COM IMPRESSORA = bridge neste PC (nao no Hostinger)

=== PC DA LOJA ===

1) Copie print_bridge INTEIRA (sem .venv):
   Abrir Print Bridge.vbs, iniciar-print-bridge.bat, verificar-bridge.bat,
   rodar-bridge.cmd, app.py, printer_core.py, printer_match.py, requirements.txt

2) Python 3.13 (ou 3.x) com Add to PATH

3) Duplo clique: verificar-bridge.bat  (opcional, diagnostico)

4) Primeira vez: iniciar-print-bridge.bat (cria .venv)
   Uso diario: Abrir Print Bridge.vbs (silencioso; log print_bridge.log)
   Diagnostico com janela: iniciar-print-bridge.bat

5) Navegador neste PC: http://127.0.0.1:9123/health  -> "ok": true

6) Site Homologacao: http://85.31.231.84:2001/LojaOnline/  Ctrl+F5
   (producao: https://pedidofacil.online/LojaOnline/)
   Configuracoes - Impressoras - mapear terminal deste PC

=== MULTIESTACAO (cada caixa / terminal) ===

Cada estacao DEVE ter:
- Print Bridge ativo (Abrir Print Bridge.vbs ou bat)
- Filas Windows instaladas (USB local + rede, se usar)
- Mapeamento COMPLETO em Configuracoes > Impressao deste terminal
  (todas as impressoras logicas com caminho Windows preenchido)

Impressoras de rede (cozinha/bar): a mesma fila deve existir / ficar
visivel no Windows de TODA estacao que distribui preparo.

Regra: comanda/conta = caminho local da estacao;
       preparo = caminho da fila de setor (geralmente rede).

Botao "Testar" na linha do terminal envia 1 pagina de prova via Bridge.

=== FORMATO ESC/POS (por impressora no cadastro) ===

- Bematech / Elgin: formato "completa" (default)
- Generica / POS80 / Generic Text Only: formato "simples"
  (ou "completa" so se testado OK nesse hardware)

Se "completa" sair com texto expandido ate o fim do papel,
troque para "simples".

=== CHECKLIST GO-LIVE POR ESTACAO ===

[ ] Bridge ativo (/health ok)
[ ] Terminal_id correto (hostname-usuario)
[ ] Todas as logicas com caminho_local
[ ] Teste comanda/conta OK
[ ] Teste preparo (cozinha/bar) OK
[ ] Duas estacoes simultaneas OK (se houver)
[ ] Rollback rapido: perfil -> simples se layout quebrar

Cenarios de validacao controlada:
- Comanda em cada estacao
- Preparo distribuido
- Impressao simultanea em 2 PCs
- Impressora desligada / caminho errado (deve falhar com mensagem clara)

=== LOGS DO BRIDGE ===

Modo silencioso (VBS): arquivo print_bridge.log nesta pasta
  (linhas [BRIDGE PRINT] / OK / ERRO com terminal_id, printer e status).

Com janela (bat): as mesmas linhas aparecem no console.

Se .venv ja criado: Abrir Print Bridge.vbs ou rodar-bridge.cmd

ERRO /d ou errorlevel no .bat = arquivo antigo quebrado.
Copie o iniciar-print-bridge.bat NOVO desta pasta.
